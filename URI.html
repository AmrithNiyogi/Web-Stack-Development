Introduction

A URI (Uniform Resource Identifier) is a string of characters used to identify a resource either on the internet or within a network. It provides a way to access and interact with resources using various protocols and methods. URIs are fundamental to the web and are used to retrieve documents, images, and other resources from web servers.

Syntax and Structure

  • Scheme: Specifies the protocol or method used to access the resource. Common schemes include:
    • http - Hypertext Transfer Protocol
    • https - Secure Hypertext Transfer Protocol
    • ftp - File Transfer Protocol
    • mailto - Email address
  • Authority: Contains the domain name or IP address of the resource and optionally a port number. This part indicates the server hosting the resource. Examples include:
    • www.example.com
    • 192.168.1.1:8080
  • Path: Specifies the exact location of the resource on the server. It is hierarchical and often resembles a file path. Examples include:
    • /images/pic.jpg
    • /articles/2024/07/30
  • Query: Provides additional parameters to refine the request. It starts with a ? and includes key-value pairs separated by &. Examples include:
    • ?search=keyword&sort=asc
    • ?user=12345&status=active
  • Fragment: Identifies a specific section within the resource. It starts with a # and is used for in-page navigation. Examples include:
    • #section1
    • #top

Example URIs

Example URI: http://www.example.com:80/path/to/resource?query=abc#section1

  • Scheme: http
  • Authority: www.example.com:80
  • Path: /path/to/resource
  • Query: query=abc
  • Fragment: section1

URI vs URL vs URN

  • URI: A broad term that includes both URLs and URNs. It provides a way to identify a resource without necessarily indicating how to access it.
  • URL (Uniform Resource Locator): A specific type of URI that provides the means to locate and access a resource. It includes the scheme, authority, and path, and often includes the query and fragment.
  • URN (Uniform Resource Name): A type of URI that provides a unique name for a resource without providing its location. It is used for persistent identifiers. Example: urn:isbn:0451450523

Best Practices for Using URIs

  • Consistency: Use a consistent naming convention and structure for URIs to ensure clarity and predictability.
  • Descriptive: URIs should be descriptive and convey meaningful information about the resource they identify.
  • Human-Readable: Design URIs to be human-readable and easy to understand, which aids in usability and debugging.
  • Versioning: Include version information in URIs when evolving APIs to maintain backward compatibility. Example: /api/v1/resources
  • Avoid Sessions: Avoid using session IDs or user-specific data in URIs to maintain statelessness and avoid exposing sensitive information.
