HTTP (Hypertext Transfer Protocol)

HTTP is a protocol used for transferring hypertext requests and information on the web. It is the foundation of data communication on the World Wide Web.

Protocol

HTTP operates on the client-server model where the client makes requests and the server provides responses. It uses TCP/IP for data transmission, ensuring reliable communication.

Characteristics

  • Stateless: Each HTTP request is independent, and the server does not store any state about the client between requests.
  • Connectionless: HTTP is a connectionless protocol, meaning each request/response pair is handled separately without persisting the connection.
  • Flexible: HTTP supports various data formats such as HTML, JSON, XML, and more.

Components

  • Requests: Initiated by the client to ask the server for resources.
  • Responses: Sent by the server in response to a client request.
  • Headers: Metadata sent along with requests and responses to provide additional information.
  • Body: Contains the main content of the request or response, such as HTML, JSON, or XML.

Request Methods

HTTP defines several methods to perform actions on resources:

  • GET: Retrieves data from the server. Example: GET /resource
  • POST: Sends data to the server to create a new resource. Example: POST /resource
  • PUT: Updates an existing resource on the server. Example: PUT /resource/1
  • DELETE: Removes a resource from the server. Example: DELETE /resource/1
  • HEAD: Retrieves headers only without the body. Example: HEAD /resource
  • OPTIONS: Describes the communication options for the target resource. Example: OPTIONS /resource
  • PATCH: Partially updates a resource. Example: PATCH /resource/1
  • TRACE: Echoes the received request for diagnostic purposes. Example: TRACE /resource
  • CONNECT: Establishes a tunnel to the server, often used for SSL. Example: CONNECT www.example.com:443

Response Format

HTTP responses include:

  • Status Line: Contains the HTTP version and status code (e.g., HTTP/1.1 200 OK).
  • Headers: Provide metadata about the response (e.g., Content-Type: application/json).
  • Body: The main content of the response (e.g., HTML, JSON).

Error Codes

Common HTTP status codes include:

  • 200 OK: The request was successful.
  • 201 Created: The request was successful and a new resource was created.
  • 204 No Content: The request was successful but there is no content to return.
  • 400 Bad Request: The server could not understand the request due to invalid syntax.
  • 401 Unauthorized: Authentication is required and has failed or not provided.
  • 403 Forbidden: The server understood the request but refuses to authorize it.
  • 404 Not Found: The requested resource could not be found.
  • 500 Internal Server Error: The server encountered an unexpected condition that prevented it from fulfilling the request.

User Authentication in HTTP

HTTP supports several authentication mechanisms:

  • Basic Authentication: Sends user credentials (username and password) encoded in base64. Example header: Authorization: Basic base64(username:password)
  • Digest Authentication: Provides a more secure method than basic authentication by using hashed credentials.
  • Bearer Token: Uses tokens (e.g., OAuth tokens) to authenticate requests. Example header: Authorization: Bearer token
  • OAuth: A more advanced protocol for authorization, allowing third-party applications to access user data without exposing credentials.
